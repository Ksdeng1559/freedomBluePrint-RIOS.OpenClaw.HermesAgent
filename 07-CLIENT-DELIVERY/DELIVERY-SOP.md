# Client Delivery SOP

## Delivery Principle

Clients do not buy code. They buy a working business outcome.

Every delivery must include:

- Live system
- Clear instructions
- Short Loom walkthrough
- Support plan
- Next improvement recommendation

## Delivery Stages

### 1. Kickoff

Collect:

- Business name
- Domain
- Brand colors
- Logo
- Offer details
- Lead qualification criteria
- Contact email
- CRM tools
- Calendar link
- Compliance/privacy considerations

### 2. Build

Follow the approved PRD only.

Do not add unapproved features.

### 3. Internal QA

Test:

- Landing page loads
- Forms validate
- Form submission works
- Database receives record
- Email notification sends
- Admin dashboard works
- CSV export works
- Mobile layout works
- Error states are readable
- Environment variables are documented

### 4. Client Review

Send:

- Live staging URL
- 2-minute Loom
- Summary of what works
- Specific feedback request
- Deadline for approval

### 5. Launch

After approval:

- Connect domain
- Confirm SSL
- Run final test lead
- Confirm notifications
- Save admin credentials securely
- Document deployment

### 6. Handoff

Provide:

- Live URL
- Admin URL
- Login instructions
- Support contact
- What to do if something breaks
- Recommended next features

### 7. Retainer Conversion

Pitch ongoing support before final handoff.

```text
I can keep this running for you, handle small updates, monitor submissions, and improve the system as your business changes.

The support plan is $[monthly]/month and includes hosting oversight, small fixes, monthly review, and direct access if anything breaks.

Would you like me to add that to the agreement?
```

## Delivery Done Definition

A project is not done until:

- Client confirms access
- A real test lead has been submitted
- Notification was received
- Dashboard displays the lead
- Handoff Loom was delivered
- Retainer was offered
- Case study/testimonial request was made
